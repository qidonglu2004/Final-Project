專案任務管理系統 - 功能規格書
版本：1.0
=====================================

【系統概述】
本系統為一套簡化版的專案任務管理系統，用以模擬 Trello / Jira 之核心功能，
提供專案管理、任務追蹤、團隊協作與進度監控等功能，
透過物件導向設計呈現任務狀態轉換、角色權限與系統擴充性。

【列舉型別（Enum）】
TaskStatus（任務狀態）

TODO（待辦）

IN_PROGRESS（進行中）

IN_REVIEW（審核中）

BLOCKED（阻塞）

DONE（完成）

Priority（優先級）
LOW
MEDIUM
HIGH
CRITICAL


【抽象類別規格】
─────────────────────────────────────
抽象類別：Task（任務）
─────────────────────────────────────
屬性：
id（字串）：任務唯一識別碼，不可為 null 或空字串
title（字串）：任務標題，不可為 null 或空字串
description（字串）：任務說明
status（TaskStatus）：任務目前狀態，預設為 TODO
priority（Priority）：任務優先級，預設為 MEDIUM
assignee（TeamMember）：被指派的成員，可為 null
statusHistory（ArrayList<StatusChange>）：狀態變更紀錄
observers（ArrayList<TaskObserver>）：觀察者清單

行為：
建構子 Task(id, title, description)
若 id 或 title 為 null 或空字串，拋出 IllegalArgumentException
訊息："[parameter] cannot be null or empty"
getStatus()：回傳目前狀態
transitionTo(TaskStatus newStatus)：狀態轉換

合法轉換規則：
TODO → IN_PROGRESS
IN_PROGRESS → IN_REVIEW, BLOCKED
IN_REVIEW → DONE, IN_PROGRESS
BLOCKED → IN_PROGRESS
DONE → IN_PROGRESS
若轉換非法，拋出 IllegalStateException

訊息："Cannot transition from [current] to [new]"
成功時：
更新狀態
新增 StatusChange 紀錄
呼叫 notifyObservers()
回傳 true 表示成功
assignTo(TeamMember member)
若 member 為 null，拋出 IllegalArgumentException
成功時設定 assignee
unassign()
清除 assignee
